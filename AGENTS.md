# Luna release rules

Every Luna application update must keep this repository consistent:

1. Update the application source and version.
2. Update the changelog and release notes in this repository.
3. Preserve all previous versions, language support, policies and documentation.
4. Never commit subscription URLs, UUIDs, API tokens, passwords, private keys or private server configuration.
5. Validate the PowerShell source, build outputs and hashes before announcing a release.
