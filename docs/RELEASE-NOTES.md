# Luna 1.5.0-release

Дата выпуска: 15.07.2026

## Split Tunneling в двух режимах

Версия 1.5.0 добавляет реальное разделение трафика в System Proxy. Luna уже
имела локальный loopback-прокси и учёт владельца соединения по PID; теперь этот
слой разбирает HTTP и CONNECT, получает полный путь процесса и выбирает прямое
подключение либо Xray upstream.

### System Proxy

- сайты и поддомены — direct по имени назначения;
- IPv4/IPv6 и CIDR — direct по адресу назначения;
- приложения и игры — direct по точному полному пути `.exe`;
- весь не совпавший proxy-aware HTTP/HTTPS трафик — через Xray;
- приложения, не использующие системный прокси, уже идут напрямую;
- UDP не перехватывается и не переключается.

### TUN

- системный TCP/UDP-трафик обрабатывается нативным TUN inbound Xray;
- работают те же правила доменов, IP/CIDR и процессов;
- для создания интерфейса требуются права администратора.

## Почему нет WFP-драйвера

Стандартные user-mode WFP-фильтры подходят для разрешения и блокировки по
приложению, но не перенаправляют произвольное соединение в локальный прокси.
Такое перенаправление требует kernel-mode callout driver, отдельного аудита и
EV/attestation signing. Для System Proxy это не нужно: приложение само
подключается к локальному прокси Luna, поэтому выбор маршрута выполняется в
user mode без ослабления безопасности Windows.

## Совместимость и приватность

- Windows 10/11 x64;
- импорт `luna.split.v1` сохранён, экспорт использует `luna.split.v2`;
- правила хранятся в `%LOCALAPPDATA%\Luna\state.json`;
- пути приложений, сайты и IP не отправляются backend Luna;
- Installer и Portable не устанавливают kernel-драйвер или постоянную службу.

## Проверки

- PowerShell 5.1 syntax и WPF XAML;
- компиляция встроенного C# proxy runtime;
- интеграционный CONNECT-тест с определением владельца по PID/EXE;
- IPv4 и IPv6 CIDR, точные и wildcard-домены;
- Xray `run -test`, Installer, Portable и SHA-256;
- production API, страницы и загрузки сайта.
