# Luna 1.4.1-release

Дата выпуска: 15.07.2026

## Исправление

Исправлено TUN-подключение серверов из JSON-подписок без исходного поля `tag`.
В версии 1.4.0 Luna ошибочно оборачивала выбранный outbound в массив и затем
пыталась назначить этому массиву служебный тег `proxy`. Xray при этом ещё не
успевал запуститься, а действие «Исправить» не могло устранить причину.

Версия 1.4.1 назначает тег непосредственно объекту outbound и сохраняет всю
реализованную в 1.4.0 функциональность Split Tunneling.

## Главное изменение версии 1.4

Split Tunneling переведён из подготовительного интерфейса в рабочий режим.
Luna использует нативный TUN inbound Xray на Windows и направляет выбранные
исключения через прямой outbound, сохраняя остальной трафик внутри VPN.

## Поддерживаемые исключения

- сайты и поддомены;
- отдельные IPv4 и IPv6 адреса;
- IPv4/IPv6 CIDR-подсети;
- приложения по полному пути к `.exe`;
- игры и их launcher по полному пути к `.exe`.

## Безопасность и приватность

- TUN запрашивает UAC только при подключении, которому нужны системные маршруты;
- правила хранятся локально в `%LOCALAPPDATA%\Luna\state.json`;
- backend Luna не получает домены, IP и пути к приложениям;
- `self/`, `xray/`, локальная сеть и исходящий физический интерфейс защищены от
  маршрутизационной петли;
- импорт принимает только схему `luna.split.v1` и проверяет домены, IP/CIDR и
  расширения `.exe`.

## Загрузки
