# Split Tunneling Luna

## Как включить

1. Откройте `Split Tunneling`.
2. Выберите область действия: `System proxy` или `TUN`.
3. Включите `Контролируемое исключение трафика`.
4. Добавьте сайты, IP/CIDR, приложения или игры.
5. Нажмите `Применить` и подключитесь к VPN. При активном соединении Luna
   безопасно перезапустит ядро, чтобы применить правила.

## System Proxy

Подходит для браузеров и программ, использующих системный HTTP/HTTPS proxy
Windows. Luna определяет PID и полный путь `.exe` соединения с локальным
прокси. Совпавшее правило идёт напрямую, остальное — через Xray.

Ограничения:

- приложение, игнорирующее системный proxy, уже работает напрямую;
- UDP не перехватывается;
- DNS для прямого доменного назначения выполняет Windows;
- IP/CIDR может потребовать разрешение имени через Windows DNS.

## TUN

Подходит, когда нужно управлять системным TCP/UDP-трафиком, включая приложения
без поддержки proxy. Luna запросит UAC при запуске TUN. Добавляйте основной
`.exe` игры и отдельный launcher, если оба создают сетевые соединения.

## Импорт и экспорт

Формат `luna.split.v2` хранит включение, область действия и четыре списка
правил. Luna также импортирует прежний `luna.split.v1`; секреты VPN в файл
Split Tunneling не включаются.

## WFP

Luna 1.5.0 не устанавливает kernel-mode WFP callout-driver. User-mode WFP
может применять permit/block по приложению, однако прозрачное connect redirect
требует подписанного callout-driver. Для System Proxy перенаправление не нужно:
proxy-aware приложение уже соединяется с Luna, а селективный proxy выбирает
маршрут по PID/EXE. Это уменьшает поверхность атаки и не требует отключения
защиты драйверов Windows.
