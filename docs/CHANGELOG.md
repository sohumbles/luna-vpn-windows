# История изменений Luna

## 1.5.0-release

- Split Tunneling теперь работает и в режиме System Proxy, без принудительного переключения на TUN;
- локальный HTTP/CONNECT-прокси Luna определяет владельца соединения по PID и полному пути `.exe`, после чего выбирает прямое соединение либо Xray upstream;
- сайты, поддомены, IPv4/IPv6 и CIDR применяются в селективном прокси; приложения и игры исключаются по полному пути;
- добавлен явный выбор области действия правил: System Proxy или TUN;
- System Proxy честно сообщает ограничения: только proxy-aware HTTP/HTTPS, без перехвата UDP; программы, игнорирующие системный прокси, уже работают напрямую;
- формат экспорта обновлён до `luna.split.v2` и сохраняет выбранную область действия; импорт `v1` сохранён;
- добавлены модульные и интеграционные тесты маршрутизации по EXE, доменам, IPv4/IPv6 CIDR и реального CONNECT-туннеля;
- неподписанный WFP kernel-driver не включён: для перенаправления proxy-aware трафика он не требуется.

## 1.4.1-release

- исправлено подключение TUN для JSON-профилей из подписок, в которых исходный proxy outbound не содержит поля `tag`;
- Luna больше не превращает выбранный outbound в массив перед назначением служебного тега `proxy`;
- добавлена регрессионная проверка конфигурации подписки без исходных тегов;
- действие «Исправить» больше не требуется для этой ошибки: конфигурация корректно формируется до запуска Xray.

## 1.4.0-release

- реализован системный режим TUN на Windows 10/11 через штатный TUN inbound Xray;
- Split Tunneling теперь реально исключает из VPN выбранные сайты, IP-адреса, IPv4/IPv6 CIDR-подсети, приложения и игры;
- для приложений и игр используются process-aware правила Xray по полному пути к `.exe`;
- добавлены отдельные списки сайтов, IP, приложений и игр, проверка ввода, удаление, сброс, импорт и экспорт правил;
- при включении Split Tunneling режим TUN выбирается автоматически, а права администратора запрашиваются только перед созданием системного туннеля;
- добавлены защита от зацикливания трафика Xray, прямой маршрут локальной сети и автоматический выбор физического исходящего интерфейса;
- правила сохраняются локально и не отправляются backend Luna.

## 1.3.5-release

- исправлено неконтролируемое обновление карточки «Задержка выбранного сервера»;
- автоматическая проверка запускается только при одновременно активном VPN-сеансе Luna и включённом переключателе «Автообновление»;
- после выключения автообновления или отключения VPN запоздавшие результаты фоновой проверки отбрасываются;
- ручная кнопка «Обновить» по-прежнему выполняет ровно одну проверку.

## 1.3.4-release

- исправлено переполнение `System.Int32` после накопления сетевыми счётчиками более 2 ГБ: дельты и суммарный трафик теперь вычисляются строго как `Int64`;
- автообновление задержки запускается только при активном VPN-соединении, созданном Luna;
- при отключённом VPN автоматические фоновые проверки задержки не выполняются, при этом ручная кнопка проверки остаётся доступной;
- результат автоматической проверки, завершившейся уже после отключения VPN, больше не обновляет интерфейс устаревшим значением;
- версия в интерфейсе, launcher, installer и MSIX приведена к единому значению `1.3.4-release`.

## 1.3.3-release

- исправлен цикл задержки: без включённого автообновления выполняется только ручная проверка;
- подпись переключателя сокращена до «Автообновление».

## 1.3.2-release

- главный экран закреплён в пределах окна: кнопка подключения находится сверху слева, а прокручивается только список серверов;
- блоки режима и задержки выбранного сервера всегда остаются видимыми без прокрутки всего экрана;
- автообновление задержки раз в 0,67 секунды больше не заменяет текущее значение надписью «Проверка…»;
- восстановлены выбор русского/английского языка и выбор темы;
- уведомление о необходимости перезапуска появляется сразу после изменения языка или темы;

## 1.3.1-release

- исправлено перепутанное отображение значений «Без VPN» и «Через VPN» в карточке «Качество маршрута»;
- логика и сетевые этапы диагностики остались без изменений;
- подготовлена отдельная поставка MSIX для обновлений через Microsoft Store;
- исправлен запуск Xray, когда путь к конфигурации содержит пробелы, например `C:\Users\Online ...`;
- действие «Исправить» подтверждает успех только после реального запуска Xray;
- успешное сообщение об очистке DNS больше не преобразуется в ошибку «DNS недоступен»;
- главная кнопка подключения перенесена влево, справа постоянно отображается список серверов;
- добавлены отдельный ping каждого сервера и кнопка «Пинг всех» с индикаторами выполнения;
- автообновление задержки перенесено на главный экран и выполняется каждые 0,67 секунды;
- режим подключения выбирается на главном экране: System proxy доступен, TUN явно отмечен как находящийся в разработке;
- сохранение локальных серверов и подписок защищено от пустого ответа каталога Luna;
- пользовательский интерфейс токена телеметрии удалён: для Luna SpeedTest токен не нужен;
- доступен Luna SpeedTest с реальными замерами ping, jitter, скачивания и отдачи.

## 1.3.0-release

- добавлена диагностика качества текущего маршрута до YouTube, Discord, Microsoft, GitHub и Cloudflare;
- реализованы отдельные фазы DNS, TCP, VPN CONNECT, TLS-handshake, HTTP, TTFB и проверка ожидаемого ответа сервиса;
- добавлено сравнение доступности без VPN и через Luna;
- проверка запускается автоматически после подключения, вручную и периодически;
- ошибки DNS, тайм-аута, TLS и HTTP отображаются без подмены результата;
- возвращено название «Текущая задержка» на главном экране;
- общая статистика сети и трафик Luna берутся из реальных системных счётчиков;
- все накопительные сетевые счётчики переведены на `Int64`, скорости — на `double`;
- внешние порты, ping и интервалы backend проходят безопасную проверку диапазона;
- сброс системного счётчика и переподключение больше не создают отрицательную скорость;
- непрерывный исторический график удалён из Desktop;
- оставлены текущая задержка выбранного сервера, время проверки, ручное обновление и один 5-секундный цикл автообновления, выключенный по умолчанию;
- быстрые повторные нажатия не запускают параллельные проверки;
- добавлен единый переиспользуемый `HttpClient` с тайм-аутами отмены;
- журнал ротируется, UI читает только хвост, локальная очередь телеметрии ограничена;
- добровольная телеметрия выключена по умолчанию, требует персональный токен и отправляет замеры пакетами;
- статистика назначений и автоматический Speedtest не имитируются и явно обозначены как недоступные;
- добавлен single-instance launcher с встроенным PowerShell runspace без внешнего окна PowerShell.

## Функции в разработке

Эти возможности пока не считаются работающими и появятся в следующих обновлениях:

- полноценный TUN-режим и Split Tunneling;
- статистика трафика по отдельным приложениям;
- Sing-box, Clash Meta, Hysteria2, WireGuard и OpenVPN;
- Kill Switch;
- расширенная защита DNS, WebRTC и блокировка DNS-утечек;
- управление IPv6;
- настоящий системный трей и автоподключение;
- автоматическая проверка обновлений;
- сбор анонимной статистики;
- определение публичного IPv4 и интернет-провайдера.
